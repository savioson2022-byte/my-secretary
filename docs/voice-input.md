# 음성 입력 기능 메모

## 현재 구현

홈 화면 입력 카드에 마이크 버튼을 추가했다. 사용자가 버튼을 누르면 브라우저의 Web Speech API를 통해 한국어 음성 인식을 시작하고, 인식된 문장을 기존 텍스트 입력창에 추가한다.

입력창에 들어간 문장은 기존 흐름을 그대로 사용한다.

1. 음성으로 생각을 말한다.
2. 인식된 문장이 입력창에 들어간다.
3. 사용자가 분류 버튼을 누른다.
4. OpenAI API 또는 fallback 규칙 기반 분류가 실행된다.
5. 분류 결과를 수정하고 저장한다.

## 진입 링크

다음 주소로 들어오면 음성 입력 안내가 표시된다.

```text
/?voice=1
```

브라우저 보안 정책상 사용자의 명시적인 탭/클릭 없이 마이크를 자동 시작하는 것은 안정적으로 보장되지 않는다. 그래서 `?voice=1`은 자동 녹음이 아니라 "음성 입력을 바로 시작할 수 있는 상태"로 안내하는 진입점이다.

## iPhone에서 추천하는 연결 방식

PWA 또는 웹앱을 홈 화면에 추가한 뒤, iPhone 단축어 앱에서 다음 동작을 만들 수 있다.

1. URL 열기 동작 추가
2. 앱 주소 뒤에 `?voice=1`을 붙인 URL 입력
3. 단축어 이름을 "나의 비서 음성 기록"처럼 지정
4. iPhone 동작 버튼, 뒷면 탭, 홈 화면 아이콘 등에 연결

## Apple Watch에서의 방향

웹앱이 Apple Watch의 물리 버튼이나 시스템 동작을 직접 제어할 수는 없다. 현실적인 방향은 Apple Watch 단축어 또는 Siri로 iPhone의 단축어를 실행해 앱의 음성 입력 진입점으로 보내는 것이다.

장기적으로 서버 DB와 계정이 연결되면, Apple Watch 단축어에서 받아쓴 텍스트를 서버 API로 보내고 앱 기록으로 저장하는 별도 경로를 만들 수 있다.

## 한계

- Web Speech API는 브라우저 지원 범위가 제한적이다.
- 일부 브라우저에서는 음성 인식이 네트워크 기반으로 동작할 수 있다.
- 마이크 권한은 사용자가 직접 허용해야 한다.
- 앱이 닫혀 있는 상태에서 물리 버튼만으로 웹 음성 인식을 자동 실행하는 것은 웹앱 단독으로 보장하기 어렵다.
