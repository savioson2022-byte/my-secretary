# 에이전트 기능 확장 메모

나의 비서에서 말하는 에이전트 기능은 사용자의 요청을 외부 서비스에서 대신 처리하는 기능이다.

예시는 다음과 같다.

- "네이버에 미용실 예약해줘"
- "쿠팡에서 생수 시켜줘"
- "내일 병원 가능한 시간 찾아줘"

## MVP에서 먼저 해야 하는 일

현재 단계에서는 바로 예약이나 결제를 실행하지 않는다. 대신 앱이 구매/예약 요청을 감지하고, 실행 전에 필요한 정보를 모아 보여준다.

필요한 확인사항은 다음과 같다.

- 구매: 상품명, 수량, 예산, 배송지, 결제 전 최종 확인
- 예약: 장소, 희망 날짜/시간, 인원, 연락처, 예약 전 최종 확인
- 공통: 외부 서비스 연결 상태, 실패 시 사용자에게 되돌려줄 메시지

## 실제 자동 실행에 필요한 구성

1. 사용자 인증과 기기 동기화
2. 요청 분류와 누락 정보 질문
3. 외부 서비스 API 또는 브라우저 자동화 연결
4. 실행 전 확인 화면
5. 실행 결과 저장
6. 실패/취소/재시도 기록

## 안전 원칙

- 결제, 예약 확정, 개인정보 제출은 사용자 확인 없이 실행하지 않는다.
- 사용자가 취소할 수 있는 단계와 로그를 남긴다.
- API 키와 서비스 계정 정보는 서버 환경변수에만 둔다.
- 쿠팡, 네이버 같은 서비스는 공식 API 또는 약관을 확인한 자동화만 사용한다.
