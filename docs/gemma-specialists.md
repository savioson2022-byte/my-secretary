# Gemma 4 전문 에이전트 구조

## 원칙

나의 비서는 Gemma 4 기본 모델을 분야마다 복제하지 않는다. iPhone과 Mac의 저장 공간과 메모리를 아끼기 위해 하나의 `Gemma 4 E2B` 기본 모델을 사용하고, 작업에 따라 작은 LoRA 어댑터와 개인 AI 메모리만 교체한다.

## 전문가

| 전문가 | 담당 작업 | 읽는 개인 메모리 | LoRA 자산 |
| --- | --- | --- | --- |
| 작업 분류 | 단기일정, 메모, 즉시처리 분류 | classification | classification.lora |
| 시간 배치 | 장소, 이동, 에너지 기반 후보 평가 | schedule, notification | schedule.lora |
| 재구매 | 상품 추출, 유사 상품, 구매 주기 | purchase | purchase.lora |
| 예약 | 예약 조건 추출, 누락 질문, 실행안 | classification, schedule | reservation.lora |
| 아이디어 | 기존 메모와 주제 연결 | idea | 기본 모델 |
| 알림 | 행동 가능한 알림 문구 | notification, schedule | 기본 모델 |

## 실행 순서

1. 입력 내용을 보고 전문 에이전트를 선택한다.
2. 해당 전문가가 필요한 개인 AI 메모리만 읽는다.
3. iPhone 앱에 모델이 설치되어 있으면 기기에서 실행한다.
4. 기기 모델을 사용할 수 없으면 기존 서버 AI를 사용한다.
5. 서버 AI도 실패하면 기존 규칙 기반 로직으로 처리한다.

## 튜닝 데이터

사용자의 원문 전체를 곧바로 학습시키지 않는다. 사용자가 수정하거나 확정한 결과를 `입력, 정답, 이유, 도메인` 형태로 축적한 뒤 검수된 예시만 전문가별 QLoRA 학습 데이터로 내보낸다. iPhone과 Mac은 같은 계정의 개인 AI 메모리를 동기화하므로 서로 다른 기기에서도 같은 판단 기준을 유지한다.

## 모델 배포

Hugging Face 토큰은 개발 중 모델 확인에만 사용하며 앱에 포함하지 않는다. 앱 설정의 `Gemma 4 설치` 버튼은 공개된 공식 LiteRT-LM 모델을 내려받는다.

- 저장 파일: `gemma-4-E2B-it.litertlm`
- 크기: `2,588,147,712 bytes`
- SHA-256: `181938105e0eefd105961417e8da75903eacda102c4fce9ce90f50b97139a63c`
- 런타임: LiteRT-LM Swift 0.13 API + iOS 0.12 공식 바이너리
- 실행 백엔드: iOS GPU, 최대 컨텍스트 4,096 tokens

다운로드 후 파일 크기와 SHA-256이 모두 일치해야 설치 완료로 처리한다. 파일이 없거나 검증에 실패하면 기기 추론을 사용하지 않고 서버 AI와 기존 규칙으로 자동 전환한다.

## iPhone 설치 순서

1. Xcode에서 새 iOS 빌드를 iPhone에 설치한다.
2. 앱의 `설정 > 기기용 Gemma 4`로 이동한다.
3. Wi-Fi와 충전 상태에서 `Gemma 4 설치`를 누르고 앱을 열어 둔다.
4. 설치 완료 후 상태가 `사용 중`인지 확인한다.
5. 홈에서 짧은 입력을 분류해 `기기의 Gemma 전문가가 분류했습니다`가 표시되는지 확인한다.

전문 LoRA 파일은 아직 학습 전이다. 현재는 전문가별 시스템 지시와 공유 메모리로 역할을 분리하며, 충분한 사용자 확정 데이터가 쌓인 뒤 `classification.lora`, `schedule.lora`, `purchase.lora`, `reservation.lora`를 배포한다.
