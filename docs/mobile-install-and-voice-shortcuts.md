# 모바일 설치와 음성 바로가기 전략

`나의 비서`는 최종적으로 사용자가 앱을 다운로드한 것처럼 빠르게 익히고,
말하기 동작으로 바로 기록하는 경험을 목표로 한다.

## 웹에서 자동으로 할 수 있는 것

- Android/Chrome 계열 브라우저의 PWA 설치 프롬프트 실행
- 현재 앱의 음성 기록 URL 복사
- Web Share API로 음성 기록 URL 공유
- iPhone 단축어 앱 열기
- 홈 화면에 추가된 상태 감지

## 웹에서 자동으로 할 수 없는 것

아래 항목은 iOS 보안 정책상 웹페이지가 사용자의 기기 설정을 직접 바꿀 수 없다.

- iPhone 홈 화면에 자동 추가
- iPhone 동작 버튼에 자동 연결
- 뒷면 탭에 자동 연결
- 단축어 자동 생성/설치
- 마이크 권한 자동 허용

따라서 앱 안에서는 버튼으로 가능한 작업을 최대한 줄이고, 불가능한 작업은
짧은 단계 안내와 템플릿 링크로 해결한다.

## 현재 구현

`/account` 페이지의 모바일 사용 준비 카드에서 다음을 제공한다.

- 설치 시작 버튼
- 음성 기록 링크 복사
- 음성 기록 링크 공유
- 단축어 앱 열기
- iPhone 홈 화면 추가 단계 안내

음성 기록 링크는 아래 형식이다.

```text
/?voice=1
```

## 최종 제품에서 권장하는 방식

1. 첫 실행 온보딩에서 설치 안내를 보여준다.
2. 로그인 이후 홈 화면 추가 여부를 확인한다.
3. 홈 화면 추가가 안 되어 있으면 기기별 안내를 보여준다.
4. 음성 기록 바로가기에는 `/?voice=1` 링크를 사용한다.
5. iPhone 사용자는 단축어 템플릿을 제공한다.
6. 동작 버튼/뒷면 탭은 사용자가 단축어를 지정하도록 안내한다.
7. Apple Watch는 별도 watchOS 앱이 없다면 iPhone 알림 미러링을 우선 사용한다.

## 더 좋은 장기 해결책

- App Store용 네이티브 iOS 앱 또는 React Native 앱을 만든다.
- App Intents/Siri Shortcuts를 붙여 "나의 비서에 기록" 음성 명령을 제공한다.
- Apple Watch 앱 또는 complication을 만들어 손목에서 바로 기록한다.
- 웹앱은 설치형 PWA와 백업 경로로 유지한다.
