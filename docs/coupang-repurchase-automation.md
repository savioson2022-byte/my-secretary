# 쿠팡 재구매 자동화 설계

## 원칙

- 쿠팡 계정 비밀번호, 결제수단, 세션 쿠키를 나의 비서 서버에 저장하지 않는다.
- 처음 사는 상품은 자동화하지 않는다.
- 이미 구매한 적 있는 상품만 재구매 후보로 표시한다.
- 상품명, 가격, 수량, 배송지, 결제수단은 사용자가 마지막에 직접 확인한다.
- 로컬 스크립트는 쿠팡 페이지를 여는 역할까지만 담당하고 결제 버튼을 대신 누르지 않는다.

## 현재 구현

1. 앱에서 구매 이력을 저장한다.
2. `에이전트위임 / 구매`로 분류된 요청이 들어오면 구매 이력과 비교한다.
3. 이력이 없는 상품은 `쿠팡에서 찾기`만 제공한다.
4. 이력이 있고 재구매 자동화가 허용된 상품은 `재구매 열기`를 제공한다.
5. `/purchase` 페이지에서 상품별 URL, 기본 수량, 최대 금액, 자동화 허용 여부를 관리한다.
6. 로컬 기기에서는 아래 명령으로 쿠팡 페이지를 열 수 있다.

```bash
npm run purchase:coupang -- --product "물티슈" --url "https://www.coupang.com/..." --quantity 1 --max-budget 20000 --confirm
```

## 추후 확장

- 쿠팡 제휴/공식 구매 API가 확인되면 서버 기반 결제 전 승인 흐름으로 전환한다.
- iPhone 단축어를 연결해 특정 상품의 재구매 페이지를 더 빠르게 열 수 있다.
- 고가 상품, 처음 사는 상품, 수량 변경, 배송지 변경은 자동화 금지 규칙을 유지한다.
